# AGENTS.md — 렉시케어 간단버전 진입점

> 이 파일은 **표지**임. 규칙·지식은 여기 두지 말고 `docs/` 하위를 가리킬 것.
> 상한 100줄. 초과 시 리팩터링해서 `docs/`로 분산할 것.

## 무엇을 만드는가

**렉시케어 간단버전** — 단어 인출 훈련 MVP.
그림(영화·사람 얼굴·사물 등)을 보여주고 → 떠오르지 않던 단어를 사용자가 기록 → 기록 적재 → 그 기록으로 훈련하는 웹앱.

- **목표**: 아주 단순한 테스트용 MVP. 시장조사 후 디자인·기능을 **쉽게 교체**하는 것이 최우선 설계 기준.
- **분위기**: 깨끗(clean). 디자인 레퍼런스는 `duolingo_design_md.pdf`.
- **스택**: Next.js + React (App Router), 프론트 위주. 백엔드 최소(브라우저 저장 우선).

## 반드시 지킬 3가지

1. **레이어 경계 준수** — `content → domain → logic → ui` 단방향 의존. 역방향 import 금지. 상세: `docs/architecture.md`.
2. **교체 용이성 우선** — 디자인은 `ui/`에, 콘텐츠는 `content/`에 격리. 시장조사 후 한 레이어만 바꿔도 되게 설계. 영리한 추상화보다 **단순·교체 가능**을 택할 것.
3. **MVP 단순성** — 묻지 않은 기능·과잉 엔지니어링 금지. 동작하는 최소 버전 먼저.

## 호출 가능한 서브에이전트 (컨텍스트 격리 목적)

- **lexi-content**: 그림→단어 카드 콘텐츠 도메인 (`content/`).
- **lexi-recall**: 훈련·기록·반복 로직 도메인 (`logic/`, `domain/`).
- **lexi-ui**: 깨끗한 UI·디자인 토큰 (`ui/`).
- **lexi-reviewer**: 읽기전용 검수 — 레이어 경계·MVP 단순성·교체 용이성.

각 스펙은 `docs/agents/index.md` 및 `.claude/agents/{이름}.md`. 역할 분담이 아니라 **컨텍스트 격리**가 목적임.

## 작업 규율

- **저작과 검수 분리**: lexi-* 가 작성하면, 같은 컨텍스트에서 자기 승인 금지. `lexi-reviewer`가 별도 패스로 검수.
- **문체**: 사용자 대면·내부 문서는 간결체("~함", "~임"). 코드 주석은 한국어 간결체.
- **디자인·프롬프트 미정**: 상세 디자인과 그림 프롬프트는 사용자가 추후 제공. 그 전까지는 자리표시자·토큰으로 비워두고 하드코딩하지 말 것.

## 절대 금지

- 역방향 레이어 import / `content`·`domain`에서 React·Next 의존.
- `utils/`, `helpers/`, `common/` 같은 의미 없는 폴더.
- 디자인 값(색·여백·폰트)을 컴포넌트에 하드코딩 — `ui/tokens`만 경유.
- 콘텐츠(그림·단어)를 코드에 하드코딩 — `content/` 데이터만 경유.
- MVP에 불필요한 백엔드·인증·DB 선도입.
