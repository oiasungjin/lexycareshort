# new-feature-slice — 레이어 가로지르는 기능 슬라이스

기능을 한 레이어에 몰아넣지 않고, 아래→위로 얇게 한 슬라이스를 만든다. 시장조사 후 교체가 쉽게.

## 1. 슬라이스 설계 (먼저 플랜)
- 무슨 기능인지 한 문장으로 정의. 불명확하면 사용자에게 확인.
- 어느 레이어가 필요한지 정함: 보통 `domain`(타입) → `logic`(동작) → `ui`(화면). 콘텐츠가 필요하면 `content`도.
- `docs/architecture.md` 경계·금지 사항 확인. MVP 단순성 우선 — 과잉 설계 금지.

## 2. 아래에서 위로 구현 (위임)
1. **domain**: 필요한 타입·규칙 추가 (lexi-recall). 순수 TS.
2. **logic**: 유즈케이스 함수 + 유닛 테스트 (lexi-recall). 정책·임계값은 인자/설정으로 빼서 교체 가능하게.
3. **ui**: 컴포넌트·화면 (lexi-ui). 디자인 값은 `ui/tokens` 경유. logic 함수만 호출, 로직 재구현 금지.
- 2개 이상 독립 작업은 병렬 위임 가능.

## 3. 검증 후 보고
- `npm run typecheck`·관련 테스트 실행(background 가능).
- `lexi-reviewer`로 별도 패스 검수(레이어 경계·교체 용이성·MVP 단순성).
- `WORKBOARD.md` 갱신. 추가 파일·교체 지점·미정 항목 보고.

## 금지
- UI에 채점·저장 로직 직접 구현. 디자인·콘텐츠 하드코딩. 한 커밋에 여러 무관 기능 섞기.
